# 更新日志 (CHANGELOG)

## [0.2.0] - 2026-05-17

### 新增
- **手动创建技能**:
  - 在新建技能弹窗中实现了“手动创建”功能。
  - 支持直接输入技能名称并编写 `SKILL.md` 的内容，后端直接在数据目录下生成相应的本地技能文件结构。
- **AI 草稿 (AI Draft)**:
  - 在新建技能弹窗中完全集成了专用的「AI 草稿」生成与预览视图。
  - 用户可填写技能名称（进行严格的命名格式校验：仅限小写字母、数字和连字符，如 `my-skill-name`）和技能描述。
  - AI 底层依据系统设定的资深 AI Agent 架构师系统提示词（System Prompt），自动解析触发场景并产出高度标准化的 `SKILL.md` 内容格式。
  - 自动生成后无缝过渡至「手动创建」预览及修改视图，允许用户自由调整生成出的草稿再最终物理保存入库。
- **扫描本地技能**:
  - 在新建技能弹窗中实现了“扫描本地”功能。
  - 接入原生的系统目录选择器 (`@tauri-apps/plugin-dialog`) 让用户快速选择已有技能目录。
  - 后端会自动检测目录下是否存在 `SKILL.md` 或 `README.md`，并在校验通过后将整个文件夹完整拷贝到应用的本地数据仓库中。
- **本地技能删除功能**:
  - 为「我的 Skills」列表卡片上的垃圾桶图标赋予了真正的删除能力。
  - 支持**即时静默删除**，移除二阶段中多余的二次确认弹窗，极速响应用户删除动作。
  - 在 Rust 后端新增 `delete_local_skill` 指令，彻底且安全地擦除本地 `my_skills` 中该技能对应的完整文件夹，并同步派发更新事件进行状态清空与自动重新统计数量。


### 修复
- **状态同步更新**: 修复了新建或导入技能后主页列表和侧边栏徽章不更新的问题。采用原生的全局 `CustomEvent('skills-updated')` 事件机制确保状态能在组件间准确传递并自动触发刷新。

## [0.1.0] - 2026-05-10

### 新增
- **设置 - AI 模型管理页面**:
  - 在「AI 模型」设置标签下新增了完整的模型管理界面。
  - 实现「端点」卡片视图，展示供应商名称、验证状态标签（已验证/未验证）、API URL 和模型数量。
  - 「测试连接」按钮点击后在卡片内显示「端点链接成功」内联提示（3 秒后自动消失）。
  - 「编辑」按钮打开「编辑端点」弹窗，支持修改供应商、API Key（可显示/隐藏）和 API 地址，点击「保存修改」实时更新端点数据。
  - 「添加模型」按钮打开「添加模型」弹窗，支持手动填写模型类型、自定义名称、供应商、API Key、API 地址、模型名称，并展示 URL 预览提示和「高级参数」折叠区。
  - 模型列表行展示模型名称、类型及各默认值标签，并提供测试、编辑、删除操作图标。
  - 数据通过 React `useState` 管理，支持运行时动态增删模型。
- **Skill 详情预览页**:
  - 新增 `/skill/:skillName` 路由用于展示技能的详细页面。
  - 在主页的卡片上增加了点击事件，点击后自动导航至详情页。
  - 详情页分为左右两栏布局。左侧为“预览”区，支持解析并使用 `react-markdown` 美观地渲染 `SKILL.md` 的内容；右侧为“平台集成”区。
  - 实现了平台集成面板的静态 UI：支持切换复制/软链接，具备全选框以及各目标平台独立的选择状态管理。
  - 在 Rust 后端新增 `get_skill_detail` 接口，用于根据路径返回完整的 `SKILL.md` 内容字符串。
- **本地 Skills 列表展示**:
  - 主页面（我的 Skills）现在支持读取并展示数据目录（`my_skills`）下已存在的 skill。
  - 为每个 skill 卡片增加了悬浮操作栏（安装到平台、添加收藏、删除）。
  - 在 Rust 后端新增了 `get_local_skills` Tauri API，用于遍历本地 skill 并从 `SKILL.md` 中提取描述信息。
- **设置专属交互视图**:
  - 新增全局设置页面（`/settings`），采用侧边导航与内容区的两栏式布局设计。
  - 新增了「常规设置」、「显示设置」、「数据设置」等系统分类侧边导航选项。
- **数据目录自定义功能**:
  - 在「数据设置」中引入了数据目录自定义功能。
  - 集成 `@tauri-apps/plugin-dialog` 实现了调用系统原生文件夹选择对话框，支持路径更新。
  - 增加了数据路径的持久化存储（LocalStorage），并自动初始化为系统的应用数据目录。
  - 配置了 Tauri v2 的 `capabilities` 权限，确保插件调用正常。
- **从 GitHub 安装技能**: 
  - 在新建技能弹窗中增加了“从 GitHub 安装”的专属交互视图。
  - 支持直接在前端输入 GitHub 仓库 URL，调用 Tauri 核心 API 在后台触发下载。
  - 在 Rust 后端新增 `clone_github_repo` 命令，默认通过 `git clone` 自动将仓库获取到设置中定义的「数据目录」下的 `my_skills` 目录中。
- **新建技能弹窗**: 实现了全新的“新建技能”对话框，支持多种添加方式。
  - **AI 草稿**: 支持通过描述需求由 AI 生成 SKILL.md。
  - **从 GitHub 安装**: 支持粘贴仓库地址进行安装。
  - **手动创建**: 支持从零开始编写技能。
  - **扫描本地**: 支持导入已有的本地技能。
- **状态管理**: 在顶部栏（Header）中集成了弹窗控制逻辑。

### 修复
- **运行时错误**: 修复了由于 `lucide-react` 缺少 `Github` 图标导致的页面白屏问题，暂时使用 `GitBranch` 替代。
- **编译优化**: 清理了 `Header.tsx` 中未使用的导入，解决了严格模式下的构建失败。
- **UI 样式**: 
  - 移除了弹窗关闭按钮及顶部栏按钮多余的黑色边框。
  - 为弹窗添加了平滑的淡入和位移动画。
  - 优化了弹窗在深色模式下的适配效果。
