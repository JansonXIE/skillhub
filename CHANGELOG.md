# 更新日志 (CHANGELOG)

## [0.2.3] - 2026-05-19

### 新增
- **已分发技能管理增强**:
  - 支持将 skill 提交 Gerrit 仓库。
  - 为已分发页面中的技能条目新增“删除”功能，支持清理不再需要的本地分发记录。
  - 实现侧边栏数字与本地存储 (`skillhub-distributed`) 的实时联动，删除操作后通过全局事件自动刷新统计数值。

### 优化
- **技能上传弹窗 (Upload Skill Modal) 深度优化**:
  - **窗体布局升级**：将上传弹窗最大宽度由 `600px` 扩展至 `800px`，显著提升长 Git URL 的展示完整性，解决排版拥挤问题。
  - **简化交互逻辑**：去除了复杂的 “Amend 模式” 勾选，统一提交路径，使 Gerrit 分发流程更直观、更防呆。
  - **UI 质感提升**：优化了字段 Label 比例及数值展示背景，确保在不同屏幕分辨率下均能保持整齐划一的视觉效果。

### 修复
- **Gerrit 提交冲突与权限问题修复**:
  - 核心逻辑改进：在执行推送前强制进行 `git reset --hard` 到远程分支头部，彻底解决了因本地缓存目录落后于远程而导致的 “change closed” 推送失败问题。
  - 提交范围隔离：将 `git add .` 精确化为针对特定技能目录的操作，防止在多技能分发场景下出现误提。
  - Change-Id 清理优化：修复了合并失败后产生残留 Commit 导致 Change-Id 冲突的 Bug，确保每次提交都是基于最新状态的洁净 PatchSet。

## [0.2.2] - 2026-05-18

### 修复
- **Tauri 构建安装包失败修复**:
  - 将 `tauri.conf.json` 中的 `identifier` 从默认值 `com.tauri.dev` 更改为 `com.skillshub.desktop`，解决了 `npm run tauri:build` 编译安装包时因 bundle identifier 不唯一而报错的问题。


## [0.2.1] - 2026-05-17

### 新增
- **全局搜索联想下拉菜单**:

  - 单个搜索匹配项支持左侧精致的字母图标徽章、上方醒目的技能名称，以及下方自动截断的描述，匹配质感高端流畅。

### 优化
- **Header 布局重构与极致居中**:
  - 重新调整了头部元素布局，利用绝对定位和 `transform` 机制，将 `Search Skill` 全局搜索框完美居中放置于 `Header` 黄金视觉中心，两侧按钮独立对齐且互不冲突，大幅提升软件门面视觉品质。
- **严格规避焦点黑框边框样式限制**:
  - 全面清理了所有输入框、联想下拉条目、新建按钮在聚焦或点击时可能会产生的原生浏览器黑色轮廓线，以透明发光与温和的渐变微动效果提供符合现代审美的无瑕交互视觉体验。

- **待分发与已分发技能管理**:

- **智能 Git 上传与 AI 自动 Commit**:
  - 在“待分发”页面的技能卡片上新增了悬停动作（Hover Action）。当鼠标滑过卡片时，会自动浮现“上传”按钮。

  - **AI 智能生成提交信息**：前端调用后端的 `prepare_skill_commit` 提取 `git diff --cached` 补丁内容，并直接读取系统中配置好的 AI 大模型端点与 API 密钥。如果检测到有效的修改，将自动向配置好的 AI 接口（如 DeepSeek, Kimi 等兼容的 OpenAI 接口）发送 diff 内容，由 AI 自动生成简短、专业、一目了然的中文 commit message。
  - 提交成功后，该技能将自动从“待分发”移入“已分发”队列，完成一次极致丝滑的自动化多平台分发流程。

  - 在技能详情页 (`SkillDetail.tsx`) 标题右侧增设了高雅的收藏按钮，支持独立点击收藏与取消收藏，与列表卡片、收藏页面数据实现完美无缝的双向同步。
  - 在侧边栏 (`Sidebar.tsx`) 接入了对自定义 `favorites-updated` 全局事件 of the window 对象的动态监听，根据加载到的本地技能列表与已收藏名单自动求取交集计算出最精确的有效收藏数徽标，并在所有页面切换或卡片操作时瞬时即时同步刷新，杜绝幽灵计数的产生。
  - **优雅无冗余的数据架构**：坚持“单一事实来源 (Single Source of Truth)”原则，抛弃了在磁盘上物理克隆“收藏目录”导致内容不一致和磁盘浪费的做法，全面通过本地极速 `localStorage` 记录收藏标识 + 自定义事件响应体系，达到了轻量化和绝对防呆、高可靠的精妙交互设计。
