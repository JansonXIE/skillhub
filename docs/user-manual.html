概述

SkillHub 是一款基于 Tauri 2.0 + React 19 构建的桌面应用程序， 专为 AI Skill（技能）的全生命周期管理而设计——从发现、创建、收藏、分发到同步， 帮助你一站式管理所有 AI Agent 技能文件（SKILL.md）。

📦

技能管理

统一管理本地所有技能，支持查看、创建、删除等操作。

⭐

收藏系统

将常用技能加入收藏夹，快速访问。

🚀

一键分发

将技能推送到 GitHub / Gerrit 远程仓库，支持 AI 自动生成 Commit 信息。

🏪

Skill 商店

从 GitHub / Gerrit 仓库浏览和导入社区技能。

🔍

全局搜索

跨页面实时搜索，支持下拉联想和高亮定位。

🔗

多平台集成

一键同步技能到 Copilot、Cursor、Gemini CLI、Trae 等 7 个主流 AI 平台。

技术栈

SkillHub 使用 React 19 + Vite 构建前端，Tauri 2.0 作为桌面桥接层，Rust 实现后端命令，支持 Windows 和 Linux。

快速开始

安装与启动

  1. 下载安装包
    从 GitHub Releases 页面下载对应平台的安装包（.msi 用于 Windows，.deb 用于 Linux）。
  2. 启动应用
    安装完成后，从桌面快捷方式或开始菜单启动 SkillHub。
  3. 配置数据目录
    首次启动后，进入 设置 → 数据设置，确认或修改数据目录路径。该目录用于存放所有本地技能文件。
  4. 创建或导入你的第一个技能
    点击右上角 「+ 新建」按钮，选择一种方式添加技能。
自动更新

SkillHub 支持 OTA 自动更新。启动时会自动检查新版本，你可以在 设置 → 关于 中管理自动检查更新开关。

我的 Skills

这是 SkillHub 的默认首页，展示数据目录中所有已导入的本地技能。每个技能以卡片形式呈现， 包含技能名称、描述文字和操作按钮。

技能卡片操作

每张技能卡片右上角有三个操作按钮（鼠标悬停时显示）：

  • 📥 安装到平台 — 将技能复制到已配置的 AI 平台目录
  • ⭐ 收藏/取消收藏 — 将技能加入或移出收藏夹（已收藏的卡片星星图标为实心蓝色）
  • 🗑️ 删除 — 从本地永久删除该技能（立即生效，无二次确认）

点击卡片主体区域会跳转到技能详情页，查看完整的 SKILL.md 内容。

注意

删除操作为立即物理删除，技能文件夹将从磁盘彻底移除，无法撤销。请谨慎操作。

创建新技能

点击右上角 「+ 新建」按钮，弹出新建技能对话框，提供四种添加方式：

1. AI 草稿（推荐）

  1. 在弹窗中选择 「AI 草稿」
  2. 输入技能名称（仅限小写字母、数字和连字符，如 my-awesome-skill）
  3. 输入技能描述，说明该技能的用途和触发场景
  4. 点击 「生成并预览」，AI 将根据你的描述自动生成标准的 SKILL.md 草稿
  5. 生成后自动进入预览编辑模式，你可以修改草稿内容
  6. 确认无误后点击 「创建技能」 保存到本地
前提条件

使用 AI 草稿功能需要先在 设置 → AI 模型 中配置至少一个 AI 端点并完成验证。

2. 从 GitHub 安装

  1. 选择 「从 GitHub 安装」
  2. 输入 GitHub 仓库 URL（如 https://github.com/owner/skill-repo）
  3. 点击 「扫描仓库」，SkillHub 会自动检测仓库结构并列出可导入的技能
  4. 从结果列表选择需要导入的技能

3. 手动创建

  1. 选择 「手动创建」
  2. 输入技能名称
  3. 在文本框中直接编写 SKILL.md 的完整内容
  4. 点击 「创建技能」 保存

4. 扫描本地

  1. 选择 「扫描本地」
  2. 系统会打开原生文件夹选择器
  3. 选择一个包含 SKILL.md 或 README.md 的本地目录
  4. SkillHub 会自动校验并将整个文件夹复制到数据目录
SKILL.md 格式规范

标准的 SKILL.md 包含两大部分：

  • YAML Front Matter（以 --- 包裹的元数据区）— 包含 name 和 description 字段
  • Markdown 主体 — 定义领域知识、执行逻辑和规则约束

技能详情页

点击技能卡片进入详情页，页面采用左右两栏布局：

左侧：预览区

  • 技能描述 — 从 Front Matter 中提取的描述信息
  • 技能内容 — 使用 Markdown 渲染的完整 SKILL.md 内容
  • AI 翻译 — 点击可将技能内容翻译为其他语言（需配置 AI 模型）
  • 复制 MD — 一键复制完整的技能 Markdown 内容到剪贴板

右侧：平台集成

  • 展示所有已在设置中启用的 AI 平台列表
  • 每个平台显示图标、名称和当前配置的目标路径
  • 勾选目标平台后点击 「批量安装」，将技能文件复制到所有选中平台的技能目录中
  • 支持全选/取消全选

支持集成的平台包括：

平台默认路径

Cursor%USERPROFILE%\.cursor\skills
Gemini CLI%USERPROFILE%\.gemini\skills
Antigravity%USERPROFILE%\.gemini\antigravity\skills

路径配置

各平台的目标路径可在 设置 → Skill 中单独自定义修改。路径支持环境变量（如 %USERPROFILE%）。

批量管理

在「我的 Skills」页面，点击 「批量管理」按钮进入批量模式：

  • 选择技能 — 点击卡片切换选中状态，也可使用「全选/清空」按钮
  • 批量收藏 — 将选中的技能批量添加或取消收藏
  • 批量同步到平台 — 打开平台选择弹窗，勾选目标平台后一键同步所有选中技能
  • 批量删除 — 永久删除所有选中的技能（操作前会弹出确认对话框）

在批量模式下，卡片左上角会显示复选框，选中状态的卡片带有蓝色高亮边框和微弱投影。点击「取消」按钮或再次点击「批量管理」可退出批量模式。

收藏

收藏页展示所有已加入收藏夹的技能。收藏状态基于 localStorage 中的收藏列表， 与「我的 Skills」和「技能详情」页中的收藏状态实时同步。

在收藏页的卡片中：

  • 点击星星按钮可取消收藏
  • 点击垃圾桶按钮可删除技能（同时移除收藏）
  • 点击卡片主体可跳转到技能详情页

侧边栏的收藏数量徽章会根据收藏列表和本地技能列表的交集自动计算，确保不会出现幽灵计数。

已分发

已分发页面展示所有已经推送到远程仓库的技能。数据来源于 localStorage 中的分发记录（skillhub-distributed）。

每张卡片底部显示：

  • 已分发至: 标签
  • 目标仓库的完整 URL（超长 URL 自动省略截断）

点击卡片右上角的垃圾桶按钮可删除分发记录（仅删除本地记录，不影响远程仓库中的内容）。已分发页面的卡片不可点击跳转，属于只读归档视图。

待分发

待分发页面展示本地存在但尚未推送到远程仓库的技能。计算逻辑为：本地全部技能 - 已分发记录中的技能 = 待分发技能。

将鼠标悬停在待分发技能卡片上，会出现 「上传」按钮，点击后进入上传流程。

上传技能到远程仓库

  1. 打开上传弹窗
    在「待分发」页面，将鼠标悬停到目标技能卡片上，点击浮现的「上传」按钮。
  2. 选择目标仓库
    从下拉列表中选择已在设置 → 代码仓库中配置的远程仓库。
  3. 提交与推送
    点击确认后，SkillHub 会：
    • 在后台创建 Git 沙箱缓存目录
    • 使用 git clone --sparse 极速拉取远程仓库
    • 将技能文件覆盖写入对应目录
    • 自动调用 AI 生成专业的 中文 Commit 信息
    • 执行 git push 推送到远程仓库
  4. 自动同步状态
    推送成功后，该技能会自动从「待分发」移入「已分发」，侧边栏数量徽章实时刷新。
Git 认证

上传功能依赖本地 Git 环境的认证配置（SSH Key 或凭据管理器）。确保你已在本机配置好对应远程仓库的访问权限。Gerrit 仓库需要使用 SSH 协议。

Skill 商店

Skill 商店让你从 GitHub / Gerrit 公共仓库中浏览和发现社区技能，支持一键导入到本地。 商店仓库管理在侧边栏中完成。

添加商店仓库

  1. 打开添加商店弹窗
    在侧边栏的「Skill 商店」分组中，点击底部的 「+ 添加商店」按钮。
  2. 选择仓库类型
    在弹窗顶部切换 GitHub / Gerrit 类型标签。
  3. 填写仓库信息
    • GitHub 模式：输入仓库 URL，如 https://github.com/anthropics/skills
    • Gerrit 模式：输入 SSH 格式的 URL，如 ssh://user@host:port/project
    • 仓库名称（可选）：自定义显示名称，留空则自动使用仓库名
  4. 确认添加
    点击确认后，SkillHub 会使用 Git 部分克隆技术快速扫描仓库结构， 自动识别技能目录并将仓库添加到侧边栏。
智能结构检测

SkillHub 会自动检测两种常见的 skill 仓库结构：含 skills/ 子目录的仓库（如 anthropics/skills）和根目录即 skill 文件夹的仓库，无需手动配置。

部分克隆策略

商店使用 git clone --depth 1 --filter=blob:none --no-checkout 仅拉取 tree 和 commit 元数据（约几十 KB），文件内容按需获取。每次打开商店时通过 git fetch --depth 1 增量更新。

浏览与导入技能

点击侧边栏中的商店仓库名称，进入该仓库的技能列表页面：

  • 技能卡片 — 每个卡片显示技能名称、描述和来源标签
  • 导入按钮 — 点击卡片右上角的下载图标，将技能导入到本地「我的 Skills」
  • 查看详情 — 点击卡片主体进入商店技能详情页，查看 SKILL.md 完整内容
  • 刷新 — 点击刷新按钮重新拉取仓库最新数据

导入成功后，技能会出现在「我的 Skills」列表中，侧边栏数量自动更新。

全局搜索

顶部栏居中的搜索框是 SkillHub 的全局搜索入口，支持根据当前页面动态切换搜索源，实现「所见即所搜」。

当前页面搜索内容源
我的 Skills所有本地技能
收藏已收藏的本地技能
已分发已分发的技能
待分发待分发的技能
Skill 商店当前选中仓库的技能

搜索交互

  • 聚焦输入框 — 自动加载当前页面范围的技能列表
  • 输入关键词 — 实时模糊匹配技能名称和描述，下拉显示联想结果
  • 点击结果 — 在当前页面高亮定位目标技能卡片（带有脉冲发光动画，持续 4.5 秒）
  • 跨页面搜索 — 如果在非列表页搜索，点击结果会自动跳转到对应列表页并触发高亮
  • 点击外部区域 — 关闭下拉菜单
  • 切换页面 — 搜索框状态自动复位，下拉列表自动关闭

设置

设置页面采用左侧导航 + 右侧内容区的布局，包含五个标签页。 点击顶部「← 返回」按钮可回到上一页面。

数据设置

管理 SkillHub 的数据存储目录：

  • 显示当前数据目录的完整路径
  • 点击 「更改」按钮打开系统原生文件夹选择器，设置新的数据目录
  • 修改后会自动触发侧边栏和技能列表的刷新
  • 默认路径为系统应用数据目录下的 SkillsHub 文件夹

AI 模型设置

配置 AI 大模型端点，用于 AI 草稿生成、Commit 信息生成、AI 翻译等功能：

端点管理

  • 支持添加、编辑端点（供应商名称、API Key、API 地址）
  • 测试连接 — 点击按钮验证端点是否可用，成功后在卡片内显示绿色「端点链接成功」提示（3 秒后消失）
  • 端点卡片显示验证状态标签（已验证 / 未验证）、API URL 和模型数量

模型管理

  • 为每个端点添加模型，配置模型名称、类型（对话模型）、自定义名称
  • 可设置模型为默认模型（用于 Quick Add、文本测试、翻译等场景）
  • 支持删除模型
兼容性

SkillHub 兼容所有 OpenAI API 格式的服务商，如 DeepSeek、Kimi、通义千问等。只需填写正确的 API 地址和 Key 即可。

Skill 平台配置

为 7 个主流 AI 平台配置技能存放路径，控制是否在技能详情页的「平台集成」面板中显示：

  • 每个平台卡片显示品牌图标和平台名称
  • 存放路径 — 显示当前配置的路径，点击「更改」通过系统文件夹选择器修改
  • 启用开关 — 勾选「在平台集成中启用」后，该平台会出现在技能详情页的集成面板中
  • 配置自动保存到 localStorage，无需手动点击保存

支持的平台：GitHub Copilot、Cursor、Gemini CLI、Antigravity、Windsurf、Trae、Qoder

代码仓库设置

配置用于分发技能的远程 Git 仓库（上传技能时的目标仓库）：

  • 添加仓库 — 填写仓库名称、选择类型（GitHub / Gerrit）、输入 Git URL 和默认分支
  • 仓库类型切换为 Gerrit 时，需额外填写 SSH 用户名
  • 已添加的仓库以卡片列表形式展示，显示仓库名称、类型标签、URL 和分支
  • 点击垃圾桶按钮删除仓库配置
  • 仓库配置数据持久化到 localStorage
仓库 URL 格式

GitHub: https://github.com/YourUsername/RepositoryName
Gerrit: ssh://user@gerrit-ai.sophgo.vip:29418/repo-name

关于

检查更新

  • 开关状态持久化到 localStorage
  • 关闭后，启动时不会触发自动更新检查（不影响手动检查）
